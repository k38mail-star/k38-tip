# 🔄 波波鸡开发工作流（SOP）

> 标准操作流程 · 三人审查制 · 无限循环改进

---

## 一、角色

| 角色 | 工具 | 职责 |
|------|------|------|
| **CC** | Claude Code | 主开发，修bug，实现功能 |
| **Kimi** | Kimi AI | 用户体验审查，UI/UX问题 |
| **小五** | Hermes/DeepSeek | 代码审查，部署，文档归档 |

---

## 二、标准循环（每轮约30-60分钟）

```
   CC提交代码 → git push
       ↓
   等待 ≤15分钟
       ↓
┌─ 三人并行审查 ─────────────┐
│ CC  → 代码逻辑/性能         │
│ Kimi → UI/UX/交互体验       │
│ 小五 → 整体质量/部署/文档   │
└──────────┬─────────────────┘
           ↓
   汇总问题 → 写入 reports/
           ↓
   小五修小问题（P2-P3），推回
           ↓
   CC修大问题（P0-P1），推回
           ↓
   GitHub Actions 自动部署
           ↓
   回到顶部，开始下一轮
```

---

## 三、时间节拍

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| 检查CC更新 | 每5分钟 | 小五/Kimi审查新commit |
| CC检查反馈 | 每15分钟 | CC查看reports/目录和issues |
| 紧急修复 | 发现P0崩溃 | 立即修，不等周期 |
| 每日归档 | 每天收工前 | 更新reports/当日总结 |

---

## 四、审查清单（每次必查）

### 🔴 P0 - 崩溃/数据错误
- [ ] 页面能否正常打开，无白屏
- [ ] API是否返回200，无500
- [ ] JS控制台无报错
- [ ] 预测标签(主胜/客胜)是否正确渲染
- [ ] 比赛无重复显示

### 🟠 P1 - 功能异常
- [ ] 搜索功能正常
- [ ] 主题切换完整
- [ ] AI智能推荐按钮可用
- [ ] 串关生成正常
- [ ] 动画播放/跳过正常

### 🟢 P2 - 体验优化
- [ ] 推荐理由文字显示
- [ ] 移动端触控区≥44px
- [ ] 加载状态有反馈
- [ ] 空数据有引导提示

### 🔵 P3 - 代码质量
- [ ] 无裸except
- [ ] 无重复变量声明
- [ ] 前后端字段名一致
- [ ] localStorage有try/catch

## 五、文档规范

### 审查报告
```
reports/YYYY-MM-DD_review.md
├── 发现问题清单（按严重度排序）
├── 已修复内容（commit + 文件）
├── 待CC处理事项
└── 环境变更日志
```

### 提交信息格式
```
类型: 简短描述

详细说明（可选）

审查人: xxx
```

类型前缀：`fix:` / `feat:` / `docs:` / `refactor:` / `chore:`

---

## 六、版本号管理

每次对线上有影响的变更，必须升级版本号。

### 版本格式

```
v主版本.次版本.修订号
例: v1.1.0 → v1.1.1 (修bug) → v1.2.0 (新增功能)
```

### 版本号位置

`app.py` 第11行：`VERSION = "v1.1.0"`

### 提交信息规范

```
类型: 描述

版本: v1.1.0 → v1.1.1
改动: 预测标签修复
审查: CC+小五
```

## 七、同步规则

**每个动作必须通知CC，方式如下：**

| 动作 | 通知方式 |
|------|---------|
| 小五修了代码 | push后GitHub Actions自动部署，reports/记录 |
| Kimi审查完成 | 报告写入reports/目录 |
| DNS/服务器变更 | 记录在reports/环境日志 |
| 版本号更新 | app.py 中改，commit信息标注 |

**CC的检查动作：**

CC每5分钟自动检查：
1. `git pull` 拉取最新代码
2. 检查 `reports/` 目录下最新的审查报告
3. 检查 `app.py` 中的 `VERSION` 是否有变化
4. 查看是否有新issue或评论
5. 处理标记为"待CC处理"的问题

---

## 八、部署链路

```
CC git push → GitHub main
    ↓
GitHub Actions（SSH_PRIVATE_KEY）
    ↓
新加坡服务器 git pull + systemctl restart
    ↓
https://boboji.beer 自动更新
```

**无需手动部署**，除非GitHub Actions挂了。

---

## 七、CC的检查动作

CC也应每15分钟做一次：
1. `git pull` 拉取最新代码
2. 检查 `reports/` 目录下最新的审查报告
3. 查看是否有新issue或评论
4. 处理标记为"待CC处理"的问题
5. 修完 push → 自动部署 → 通知大家审查
