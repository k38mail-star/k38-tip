# 👁️ 视觉测试方案

> 每次部署自动截图验证UI，有问题立刻回滚，不等用户发现

---

## 流程

```
CC push → GitHub Actions → 部署到新加坡
                          → Playwright打开页面截图
                          → 检查卡片数量/结构/预测标签
                          → 通过 → 上传截图到Artifacts
                          → 失败 → 上传失败截图 + 报警
```

## 检查项

| 检查 | 失败则 | 截图 |
|------|--------|------|
| 比赛卡片数量>0 | ❌ 部署失败 | 失败截图 |
| 每张卡片有复选框+球队名+胜率 | ❌ 部署失败 | 失败截图 |
| 预测标签(mc-rec)存在 | ⚠️ 警告 | 通过截图 |
| 控制台无404/500资源错误 | ⚠️ 警告 | 通过截图 |

## 截图查看

每次部署后去 GitHub → Actions → 最新一次workflow → Artifacts → `screenshot-pass` / `screenshot-fail`
